## 2026-03-19 (2) 핵심 기능 구현

**시간**: 오후 15:00 ~ 18:00 (CST)
**주제**: 전체 기능 구현

### 구현 내역

1. **기본 구조**: 비밀번호 인증, 파일시스템 스캐너, 사이드바 네비게이션
2. **칸반 + 리스트 뷰**: Kanban/List 전환, 드래그앤드롭 단계 이동
3. **Ideas 페이지**: 아이디어 전용, Promote/Discard
4. **메타태그**: YAML frontmatter 전환 (15개 파일), 중요도/위급도/긴급도/협업/오너
5. **진행률**: 서브태스크 + 4등분 바, 목표/실제 종료일
6. **문서 관리**: MD 에디터, 다중 선택 삭제, 새 파일 생성
7. **작업지시**: 단계 전환 자동 생성, 수동 생성, Claude Code 연동
8. **Work Execution**: 임베디드 터미널 (xterm.js + WebSocket PTY)
9. **Work Status**: 전체 프로젝트 작업현황 대시보드
10. **People**: 관계자 카드, connections, Related People 태그
11. **Quick Note**: _temp/ 연동, 카테고리 이동
12. **Trash**: 복원/영구삭제

### 아키텍처 결정

- DB 없음 — 파일시스템 직접 스캔 (JSON 저장: auth, tokens, people, card_order)
- YAML frontmatter 표준화 (blockquote에서 마이그레이션)

- 작업지시 프롬프트에 "체크리스트 업데이트하라" 자동 포함 (자동완료 대신)
- 파일 기반 토큰 (서버 재시작 시 유지)

### 발견된 이슈

- _find_project_path 잔여 폴더 문제 (이동 후 빈 폴더 남음)

## 2026-03-21 (1) 집중모드(Focus Mode) 기능 제안 평가

**시간**: 저녁
**주제**: 대시보드 집중모드 기능 설계 검토

### Chad 제안 원안

- 대시보드 상단 바, Dashboard 옆에 "집중모드" 버튼 추가 (다국어 지원)
- 클릭 시 활성화: 버튼 색상 변경, 사이드바 자동 접힘
- 모든 아이디어/프로젝트 카드가 흐리게 표시 + 체크박스 활성화
- 체크박스에 체크하면 해당 카드만 밝아지고, 나머지는 흐린 채 클릭 불가
- 단일 카드만 선택 가능

### MoAI 평가

**긍정적 측면:**
- 멀티 프로젝트 환경에서 주의 분산 방지 효과가 명확
- 물리적 데스크의 "하나만 펼치고 나머지 치우기"를 디지털로 구현한 컨셉
- 토글 방식이라 진입/퇴출 비용이 낮음

**우려 사항:**
1. 체크박스 UX 중복 — 흐려진 카드를 직접 클릭하면 되는데 별도 체크박스 필요성 불명확
2. 단일 선택 제한 — 관련 프로젝트 2-3개를 동시에 참조해야 하는 실무 케이스 빈번
3. 사이드바 자동 접기 — 집중모드에서도 네비게이션이 필요한 경우 있음. 사용자 선택에 맡기는 것이 나음
4. 상태 유지 — 새로고침/페이지 이동 후 복귀 시 집중모드 풀리면 사용성 저하

### MoAI 대안 제안

- 체크박스 제거 → 흐려진 카드 직접 클릭으로 선택/해제 (토글)
- 복수 선택 허용 (1~3개)
- 사이드바 접기는 옵션으로 분리 (자동 접기 OFF 가능)
- localStorage에 집중모드 상태 저장 (새로고침 유지)

### 결정 사항 (1차)

- 체크박스 제거 → 카드 직접 클릭 선택/해제 (토글)
- 최대 3개까지 복수 선택 허용
- 사이드바 자동 접기 취소 (현상 유지)
- localStorage에 집중모드 상태 + 선택 카드 ID 저장
- 버튼 재클릭 시 전체 원상복구 (버튼 색상 + 카드 흐림 해제)

## 2026-03-21 (2) 집중모드 v2 수정안 상세 평가

**시간**: 저녁
**주제**: Chad 수정안에 대한 MoAI 상세 평가

### 수정안 요약

| 항목 | 원안 | 수정안 |
|------|------|--------|
| 선택 방식 | 체크박스 | 카드 직접 클릭 |
| 선택 수 | 1개 | 최대 3개 |
| 사이드바 | 자동 접기 | 유지 (변경 없음) |
| 상태 저장 | 없음 | localStorage |
| 해제 | - | 버튼 재클릭 → 전체 원상복구 |

### MoAI 상세 평가

**긍정:**
- 체크박스 제거로 인터랙션 단순화
- 3개 복수 선택은 "메인 + 관련 프로젝트" 패턴을 자연스럽게 지원
- 사이드바 유지는 올바른 판단

**추가 고려 필요 사항 6가지:**

1. 개별 해제 — 밝은 카드 재클릭 시 개별 해제 토글 필요 여부
2. 3개 초과 클릭 시 — 무반응보다 toast "최대 3개" 안내가 UX상 나음
3. 진입 초기 상태 — 모든 카드 흐림 + 아무것도 선택 안 된 상태로 시작
4. localStorage 키 설계 — focusMode: boolean + focusedCards: string[] 저장
5. 시각적 구분 — blur보다 opacity-30이 나음 (내용 인식 가능하면서 비활성 명확)
6. 선택 표시 — 밝아진 카드에 ring 테두리 또는 좌상단 번호 배지(1,2,3) 추가

### 종합 의견

수정안은 원안의 우려를 모두 해소. 구현 복잡도 낮음. 위 6가지 디테일만 결정하면 구현 가능.

### 결정 사항 (2차)

1. **개별 해제**: 채택 — 밝은 카드 재클릭 시 개별 해제 (토글 방식)
2. **초과 클릭 피드백**: 채택 — 4번째 카드 클릭 시 toast "최대 3개까지 선택 가능"
3. **초기 상태**: 확정 — 진입 시 모든 카드 흐림 + 아무것도 선택 안 된 상태
4. **localStorage 키 설계**: 확정 (아래 상세)
5. **시각 표현**: opacity-30으로 1차 구현 후 테스트, 필요시 조절
6. **선택 표시**: 좌상단 번호 배지(1, 2, 3) 채택

### localStorage 키 설계 상세 (항목 4)

저장 구조:
```json
{
  "pm_focus_mode": true,
  "pm_focus_cards": ["project-name-1", "project-name-3"]
}
```

- `pm_focus_mode` (boolean): 집중모드 활성화 여부. `true`면 페이지 로드 시 자동으로 집중모드 진입
- `pm_focus_cards` (string[]): 선택된 카드의 프로젝트명 배열. 최대 3개. 선택 순서 유지 (배열 인덱스 = 배지 번호)

동작 시나리오:
- 집중모드 ON + 카드 클릭 → 배열에 push (최대 3개)
- 밝은 카드 재클릭 → 배열에서 제거, 나머지 배지 번호 재정렬
- 집중모드 버튼 재클릭 → `pm_focus_mode: false`, `pm_focus_cards: []` 초기화
- 새로고침 → localStorage에서 읽어서 집중모드 + 선택 카드 복원
- 다른 페이지 이동 후 대시보드 복귀 → 동일하게 복원
